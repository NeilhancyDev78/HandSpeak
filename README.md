## Overview

HandSpeak is a real-time American Sign Language (ASL) interpreter that translates hand gestures into text and synthesised speech using only a standard webcam. It runs entirely on-device — no internet connection, no cloud API, no latency from round-trips.

The system is built around a five-layer temporal recognition engine that eliminates the false positives and missed commits that plague simpler debounce approaches. A fully custom data collection suite — rendered in an animated Dr. Strange–inspired interface — lets you register, update, and manage gesture datasets without touching a single line of code.

---

## Feature Highlights

- **Threaded hand tracking** — MediaPipe detection runs on a dedicated daemon thread; the render loop never blocks
- **Temporal voting engine** — majority vote window + confidence gate + hold requirement + break detection + cooldown, all independently tunable via config
- **Repeat-letter support** — the same letter can be signed consecutively; the engine detects the inter-sign break and re-commits cleanly
- **Dr. Strange collector UI** — fullscreen Pygame interface with animated arcane mandalas, gesture index panel, and arcane countdown rings
- **Surgical dataset updates** — re-collect samples for one specific gesture without touching any other label's data
- **Config-driven** — zero magic numbers in code; all thresholds, cooldowns, camera settings, and UI parameters live in `config/`
- **Non-blocking TTS** — text-to-speech dispatched to a per-utterance daemon thread; never stalls the video pipeline
- **Adaptive inference throttle** — inference rate scales with live FPS to maintain smoothness on lower-end hardware

## Recognition Engine

The core of HandSpeak is a five-layer commit pipeline in `core/gesture_engine.py`. A raw per-frame prediction only becomes a committed letter after passing all five gates in sequence:

```
Frame prediction
      │
      ▼
[1] Confidence Gate      min_confidence = 0.80 — low-certainty frames discarded
      │
      ▼
[2] Majority Vote        vote_window = 12 frames, threshold = 75% consensus
      │
      ▼
[3] Hold Requirement     gesture must be stable for 15 consecutive frames
      │
      ▼
[4] Break Detection      repeated letter requires hand absence for 400ms
      │
      ▼
[5] Commit Cooldown      minimum 1500ms between any two commits
      │
      ▼
  ✦ COMMITTED
```

All thresholds are tunable in `config/gesture_config.json` under the `recognition` block.

## Setup Guide

### Prerequisites

| Requirement | Version |
|---|---|
| Python | 3.10 or higher |
| Webcam | Any USB or built-in camera |
| OS | Windows 10/11, macOS 12+, Ubuntu 20.04+ |

> **Windows users:** Install [Microsoft C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) if TensorFlow installation fails.

---

### Step 1 — Clone the Repository

```bash
git clone https://github.com/your-username/HandSpeak.git
cd HandSpeak
```

---

### Step 2 — Create a Virtual Environment

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# macOS / Linux
python3 -m venv .venv
source .venv/bin/activate
```

---

### Step 3 — Install Dependencies

```bash
pip install -r requirements.txt
```

---

### Step 4 — Download the MediaPipe Hand Landmarker Model

HandSpeak uses MediaPipe's `hand_landmarker.task` for skeletal landmark detection. This file is not bundled in the repository due to its size (~8MB).

```bash
# Windows (PowerShell)
Invoke-WebRequest -Uri "https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/latest/hand_landmarker.task" -OutFile "hand_landmarker.task"

# macOS / Linux
curl -L -o hand_landmarker.task \
  "https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/latest/hand_landmarker.task"
```

Place `hand_landmarker.task` in the **project root** (same level as `main.py`).

---

### Step 5 — Collect Gesture Data

Launch the collector application:

```bash
python collector/app.py
```

The fullscreen interface presents two options:

**Register New Datasets**
Walks through every gesture label (A–Z + SPACE, DELETE, SPEAK) in sequence. For each gesture:
1. The side panel shows which labels are locked ✦ and which are pending ○
2. Press `SPACE` to begin — a 3-second arcane countdown starts
3. Hold your hand sign steady — 200 samples are collected automatically
4. A 2-second cooldown follows before the next gesture begins
5. Press `R` at any time to redo the current gesture

> If you have already collected data for some gestures in a previous session, those labels are automatically skipped.

**Update a Dataset**
Select any individual gesture from the grid (arrow keys or click), confirm, and re-collect only that gesture's samples. All other labels remain untouched.

---

### Step 6 — Train the Model

```bash
python train.py
```

This will:
- Load all `.npy` arrays from `data/`
- Train a 3-layer dense network (256 → 128 → 64) with BatchNorm and Dropout
- Run EarlyStopping with patience 10 on validation accuracy
- Save the trained model to `model/gesture_model.keras`
- Write a per-class classification report to `model/training_report.txt`

Expected training time: 2–5 minutes on CPU for a full 29-gesture dataset.

---

### Step 7 — Launch HandSpeak

```bash
python main.py
```

| Control | Action |
|---|---|
| Sign a letter | Commits to text after stable hold |
| `D` key / DELETE button | Remove last character |
| `S` key / SPEAK button | Read current text aloud |
| `C` key / CLEAR button | Wipe the text buffer |
| `Q` / `ESC` | Quit |

## Configuration Reference

### `config/gesture_config.json` — Recognition Tuning

| Key | Default | Description |
|---|---|---|
| `vote_window_frames` | `12` | Frames in the majority-vote sliding window |
| `vote_threshold_pct` | `0.75` | Minimum consensus fraction to proceed |
| `min_confidence` | `0.80` | Per-frame model confidence floor |
| `hold_frames_required` | `15` | Stable frames needed before commit eligibility |
| `break_timeout_ms` | `400` | Hand-absence duration required before repeat commit |
| `commit_cooldown_ms` | `1500` | Minimum gap between any two commits |
| `samples_per_gesture` | `200` | Samples collected per gesture during registration |
| `pre_capture_countdown` | `3` | Countdown seconds before capture begins |
| `post_capture_cooldown` | `2` | Cooldown seconds after capture completes |

### `config/app_config.json` — System Settings

| Section | Key | Description |
|---|---|---|
| `camera` | `index` | Camera device index (0 = default) |
| `camera` | `width / height` | Capture and display resolution |
| `inference` | `infer_resolution` | Downscale resolution fed to MediaPipe |
| `inference` | `detection_confidence` | MediaPipe hand detection threshold |
| `ui` | `show_fps / show_confidence` | Toggle HUD elements |
| `tts` | `rate / volume` | Speech rate (WPM) and volume (0.0–1.0) |
| `performance` | `target_fps` | FPS target for adaptive throttle |

## Customising Gestures

To add, remove, or rename a gesture:

1. Edit `config/gesture_config.json` — add or remove entries under `"gestures"`
2. Re-collect data via `python collector/app.py` → **Register New Datasets**
3. Re-train via `python train.py`

The label order is alphabetically sorted at runtime by `GestureManager` and is guaranteed consistent between training and inference.

## Model Architecture

```
Input  (63,)          — 21 landmarks × (x, y, z), wrist-normalised
Dense  256  ReLU
BatchNorm + Dropout 0.35
Dense  128  ReLU
BatchNorm + Dropout 0.30
Dense   64  ReLU
BatchNorm + Dropout 0.20
Dense   N   Softmax   — N = number of gesture classes
```

Optimiser: Adam (lr=1e-3) — Loss: Sparse Categorical Crossentropy

## Troubleshooting

**Camera not opening**
Verify `camera.index` in `app_config.json`. Try `1` or `2` if `0` doesn't work.

**Hand detected but nothing commits**
Lower `min_confidence` or `vote_threshold_pct` in `gesture_config.json`. Ensure adequate lighting and a plain background.

**Same letter only commits once**
This is expected — a brief hand withdrawal (400ms, configurable via `break_timeout_ms`) is required between repeated letters.

**TTS silent on Windows**
Ensure SAPI5 is enabled. Check that no other application has an exclusive hold on the audio device.

**Low FPS**
Lower `infer_resolution` in `app_config.json` (e.g. `[320, 240]`). The adaptive throttle will also reduce inference frequency automatically.
